# 배포 가이드 — GitHub + Streamlit Community Cloud

이 문서는 `웹앱_streamlit` 폴더를 GitHub에 올리고, Streamlit Community Cloud에 배포해서
**인터넷 어디서든 링크만 있으면 접속 가능한 웹앱**으로 만드는 방법을 설명합니다.

업로드한 파일(엑셀/PDF)은 각 사용자의 브라우저 세션에서만 메모리에 보관되며,
다른 사용자와 공유되지 않습니다. (서버에 영구 저장되지 않음)

---

## 0. 준비물

- GitHub 계정 (없으면 https://github.com 에서 무료 가입)
- Streamlit Community Cloud 계정 (https://share.streamlit.io — GitHub 계정으로 로그인 가능)

---

## 1. GitHub 저장소(Repository) 만들기

1. https://github.com 에 로그인 → 우측 상단 **+** → **New repository** 클릭
2. Repository name 입력 (예: `defect-rate-webapp`)
3. **Public** 또는 **Private** 선택
   - Streamlit Community Cloud 무료 플랜은 Public 저장소 권장 (Private도 가능하나 연동 설정 필요)
4. **Create repository** 클릭

---

## 2. `웹앱_streamlit` 폴더 업로드

### 방법 A — 웹 브라우저로 업로드 (Git 설치 불필요, 추천)

1. 방금 만든 저장소 페이지에서 **Add file → Upload files** 클릭
2. `웹앱_streamlit` 폴더 안의 모든 파일/폴더를 끌어다 놓기
   - `app.py`, `pages/`, `core/`, `requirements.txt`, `packages.txt`,
     `.streamlit/`, `plan_budget.xlsx`, `표준불량명칭.xlsx`, `납품업체_브랜드_정리.xlsx`, `.gitignore`
   - ⚠️ `.streamlit/secrets.toml` 파일은 절대 업로드하지 마세요 (있다면). `secrets.toml.example`만 올리면 됩니다.
3. 하단 **Commit changes** 클릭

> 참고: GitHub 웹 업로드는 폴더째로 드래그하면 폴더 구조가 그대로 유지됩니다.
> 폴더가 인식되지 않으면, 폴더를 압축(zip)하지 말고 폴더 안의 파일들을 직접 선택해 업로드하세요.

### 방법 B — Git 사용 (터미널 사용 가능한 경우)

```bash
cd "웹앱_streamlit"
git init
git add .
git commit -m "Initial commit: Streamlit 웹앱"
git branch -M main
git remote add origin https://github.com/<본인계정>/<저장소명>.git
git push -u origin main
```

---

## 3. Streamlit Community Cloud 배포

1. https://share.streamlit.io 접속 → GitHub 계정으로 로그인
2. **Create app** (또는 **New app**) 클릭
3. 다음 정보 입력:
   - **Repository**: 방금 만든 저장소 선택 (예: `본인계정/defect-rate-webapp`)
   - **Branch**: `main`
   - **Main file path**: `app.py`
     - 만약 저장소 루트가 아니라 `웹앱_streamlit` 폴더 안에 있는 경우: `웹앱_streamlit/app.py`
4. **Deploy** 클릭

배포가 시작되면 자동으로:
- `requirements.txt`에 명시된 Python 패키지 설치
- `packages.txt`에 명시된 시스템 패키지(`fonts-nanum`, 한글 폰트) 설치

5분 내외로 빌드가 완료되면, `https://<앱이름>-<랜덤문자>.streamlit.app` 형태의 링크가 생성됩니다.
이 링크를 누구에게나 공유하면 바로 접속 가능합니다.

---

## 4. (선택) AI 코멘트 기능 활성화 — ANTHROPIC_API_KEY

공장·지역 분석 탭의 "코멘트 생성" 버튼은 `ANTHROPIC_API_KEY`가 없어도
규칙 기반 코멘트로 정상 동작합니다. Claude API를 이용한 더 자세한 코멘트를 원하면:

1. Streamlit Cloud 앱 화면 → 우측 하단 **⋮ (점 3개) → Settings → Secrets**
2. 아래 내용을 입력 후 저장:

```toml
ANTHROPIC_API_KEY = "sk-ant-여기에-실제-키-입력"
```

3. 앱이 자동으로 재시작되며 적용됩니다.

---

## 5. 데이터 파일 업데이트 방법

다음 3개 파일은 `웹앱_streamlit` 폴더에 포함되어 배포되며, 앱의 "기본값"으로 사용됩니다.
(사용자가 화면에서 직접 업로드하면 그 파일이 우선 사용됩니다)

- `표준불량명칭.xlsx` — 불량명 표준화 매핑 기준
- `납품업체_브랜드_정리.xlsx` — 브랜드 정규화 매핑표
- `plan_budget.xlsx` — 실적 목표예산

이 파일들을 갱신하려면, GitHub 저장소에서 해당 파일을 다시 업로드(덮어쓰기)하면
Streamlit Cloud가 자동으로 재배포합니다.

---

## 6. 로컬에서 먼저 테스트하기

배포 전 로컬에서 동작을 확인하려면:

```bash
cd 웹앱_streamlit
pip install -r requirements.txt
streamlit run app.py
```

브라우저에서 `http://localhost:8501` 로 접속됩니다.

## 7. 문제 해결 (Troubleshooting)

| 증상 | 원인 / 해결 |
|---|---|
| PDF 보고서에 한글이 깨짐(□□□) | `packages.txt`에 `fonts-nanum`이 포함되어 있는지 확인. Streamlit Cloud는 자동 설치하지만, 반영까지 앱을 한 번 재부팅(Reboot app) 필요할 수 있음 |
| `ModuleNotFoundError` | `requirements.txt`에 해당 패키지가 빠졌는지 확인 후 추가 → 자동 재배포됨 |
| 업로드한 파일이 새로고침 후 사라짐 | 정상 동작입니다. 모든 업로드/분석 결과는 세션(브라우저 탭) 단위로만 메모리에 보관되며, 영구 저장되지 않습니다. 결과는 각 화면의 다운로드 버튼으로 저장하세요 |
| 앱이 "Zzz... App is sleeping" 상태 | 무료 플랜은 일정 시간 미사용 시 잠자기 상태가 됩니다. 링크 접속 시 자동으로 다시 깨어납니다(약 30초~1분 소요) |
